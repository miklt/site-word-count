['', 'A interface LAG constituí o recurso que combina múltiplas interfaces físicas em única interface lógica, a qual chamamos de “LAG (Link Aggregation Group)” e tem como uma de suas principais finalidades aumentar a largura de banda do tráfego da rede para o total da largura de banda das interfaces físicas agregadas, proporcionando maior largura de banda bem como redundância de link, no caso de um dos links falhar, mantendo automaticamente o tráfego para a(s) outra(s) interface(s) do LAG.O LACP (Link Aggregation Control Protocol), é um protocolo de negociação de camada 2, que fornece métodos para controlar a agregação de várias portas físicas em um único grupo lógico. Ele permite que um dispositivo de rede possa negociar um agrupamento automático de links enviando os pacotes LACP para outro dispositivo diretamente conectado que também implemente o LACP.', 'Para adicionar qualquer interface do tipo LAG é necessário, que duas ou mais interfaces estejam disponíveis. Caso contrário durante o processo de criação, elas não serão exibidas no painel Aggregate interfaces.', 'O Blockbit UTM disponibiliza esse recurso em 3 (três) modos de operação:Modo Aggregation;Modo Balance;Modo Active/Backup.Para configurar o serviço de agregação de links, precisamos criar um grupo de agregação “LAG - Link Aggregation Group”, e definir quais links “interfaces” serão membros do grupo.', 'Para configurar uma interface “LAG” as interfaces precisam estar “desabilitadas”[].', 'A seguir vamos analisar os modos de operação mais profundamente, começando pelo Modo Aggregation.Para mais informações sobre como adicionar outros tipos de interface, acesse esta\xa0página.', '']
