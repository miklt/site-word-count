['', 'Este comando serve para exibir os logs de debug do serviço de SD-WAN efetuado no tempo estipulado na opção “Intervalo de monitoramento” no painel de criação de um perfil SD-WAN (vide capítulo\xa0SD-WAN). Há dois tipos de log disponíveis: Targets e Results e é possível levantar um log baseado em uma data específica.TargetsExibe o log de debug para todos os alvos de monitoramento. É composto de:date: A data e o horário de execução do log. A data é disposta no formato Ano-Mês-Dia. Ex.: 2019-02-12 10:02:09;profile: O nome do perfil SD-WAN de onde esse log foi gerado. Ex.: QA-Internet;dev: A interface web de onde esse log foi gerado. Ex.: eth0;target: O alvo de monitoramento que gerou esse log. Ex.: 8.8.8.8;port: A porta de saída utilizada. Ex.: 80;protocol: O protocolo que foi utilizado. Ex.: tcp;latency: O resultado do indicador de performance de Latência. É disposto em milissegundos. Ex.: 9.42;jitter: O resultado do indicador de performance de Jitter. É disposto em milissegundos. Ex.: 7.54;packet_loss: O resultado do indicador de performance de Perda de Pacotes. É disposto em números inteiros. Ex.: 0;bandwidth: O resultado do indicador de performance de Limite de Banda. É disposto em bytes por segundo. Ex.: 467711.73;connection_error: Representa todos os erros de conexão. É disposto em números inteiros. Ex.: 0;total_error: Representa a quantidade de vezes que um teste de disponibilidade do link (utilizando os valores determinados pelos indicadores de performance) retornou um erro e também os erros de conexão. É disposto em números inteiros. Ex.: 0.ResultsExibe o log de monitoramento de acordo com o resultado dos testes dos indicadores de performance. É composto de:date: A data e o horário de execução do log. A data é disposta no formato Ano-Mês-Dia. Ex.: 2019-02-10 10:25:07;profile: O nome do perfil SD-WAN de onde esse log foi gerado. Ex.: QA-Internet;dev: A interface web de onde esse log foi gerado. Ex.: eth1;latency: O resultado do indicador de performance de Latência. É disposto em milissegundo. Ex.: 12.835;jitter: O resultado do indicador de performance de Jitter. É disposto em milissegundo. Ex.: 1.0275;packet_loss: O resultado do indicador de performance de Perda de Pacotes. É disposto em porcentagem. Ex.: 0;bandwidth: O resultado do indicador de performance de Limite de Banda. É disposto em bytes. Ex.: 1818958.07;total_error: Representa a quantidade de vezes que um teste de disponibilidade do link (utilizando os valores determinados pelos indicadores de performance) retornou um erro e também os erros de conexão. É disposto em números inteiros. Ex.: 0;total_error_percent: Representa a porcentagem de falha (Equivale ao campo Fail Rate da interface visual). É disposto em porcentagem. Ex.: 0;weight: Peso do consumo da rede pela interface web. É disposto em porcentagem. Ex.: 100.00;status: O estado desta interface, se ela está on ou off. Ex.: on.Modo de uso:Command Line Interface – debug-sdwanExemplo 1: Utilizando o tipo de log “targets” em um perfil específico:Command Line Interface – debug-sdwan -i targetsExemplo 2:\xa0Utilizando o tipo de log “results” em um perfil específico:Command Line Interface – debug-sdwan -i resultsExemplo 3: Utilizando o tipo de log “targets” em um determinado perfil dentro de uma data específica:Command Line Interface – debug-sdwan -s', '']
