['', 'Habilite os protocolos dinâmicos, baseados nas características de cada protocolo e na sua estrutura de rede, sua função é otimizar suas tabelas de roteamento, melhorar o desempenho do tráfego de rede e a performance do tratamento de pacotes.Para habilitação dos protocolos de roteamento dinâmico,\xa0considere as informações à seguir:Dynamic Routing[] OSPF:\xa0Ao marcar a\xa0caixa de checagem\xa0[] o OSPF será habilitado.\xa0O\xa0Open Shortest Path First (RFC 3101)\xa0é um protocolo que se utiliza do LSR (Link State Routing) para efetuar o roteamento. Basicamente os roteadores se comunicam e divulgam entre si o estado atual de seu link, utilizando-se desta informação, um banco de dados é criado e através dele os roteadores determinam e priorizam o caminho mais curto disponível entre ele mesmo e outro roteador;[] BGP:\xa0Ao marcar a\xa0caixa de checagem\xa0[] o BGP será habilitado. O Border Gateway Protocol (RFC1105) (incluindo o Multicast Border Gateway Protocol - MP-BGP [RFC4760]) atua através das determinações configuradas por um administrador de rede, gerencia o roteamento de pacotes e os direciona entre Sistemas Autônomos (AS).\xa0É um dos protocolos padrão de gateway externo e de roteamento entre domínios;[] RIP:\xa0Ao marcar a\xa0caixa de checagem\xa0[]\xa0o RIP será habilitado.\xa0O Routing Information Protocol (RFC 1850) é um protocolo de roteamento de vetor de distância que age se comunicando com os roteadores mais próximos e enviando informações quanto à distância da rede constantemente, através dessas atualizações periódicas é feita a convergência para determinada topologia;[] PIM-SM:\xa0Ao marcar a\xa0caixa de checagem\xa0[] o PIM-SM\xa0será habilitado.\xa0Significa Protocol Independent Multicast - Sparse Mode (RFC 7761), não possui nativamente meios de descobrir topologias e não cria tabelas de roteamento, porém é capaz de se utilizar da informação de outros protocolos de roteamento para esse fim e cria árvores compartilhadas de tráfego multicast, sendo capaz de usá-las para definir o caminho mais curto por origem. O PIM-SM, se utiliza de reverse path forwarding (RPF) para comunicação com grupos multicast sendo que seus pacotes se subdividem conforme novos nodos são introduzidos ao grupo, nenhum encaminhamento é feito sem requisição e para localizar outra fonte de tráfego multicast é criado um ponto de encontro na rede (RP - Redezvous Point).Para visualizar [Exemplos] de configuração, na interface de habilitação de roteamento dinâmico, clique em HELP [] para cada protocolo respectivamente.Para configurar os protocolos de roteamento dinâmico, acesse o console SSH através da interface “Web” ou por meio de um cliente SSH. Ex.: “Putty”, “MobaXterm”.
