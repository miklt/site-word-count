['', 'O Blockbit UTM suporta adicionar interfaces virtuais do tipo “MAC VLAN”, e requer obrigatoriamente uma interface física [EthX].Uma interface de rede pode ter vários endereços IP’s, inclusive para segmentos de redes lógicas diferentes, mas não isolam o tráfego, que pode ser “identificado” ou “interceptado” nesta interface.Em uma MAC VLAN é assumido uma única interface de rede e permite criar múltiplos endereços virtuais com diferentes MAC, ou seja, “N para 1”. Uma MAC VLAN só poderá ver o tráfego que tenha um endereço MAC correspondente à interface, evitando que outras interfaces virtuais “identifiquem” ou “interceptem” o tráfego destinado a outra MAC VLAN.Para configurar uma interface Virtual,\xa0 siga os passos a seguir:Clique em []\xa0e selecione a opção\xa0[VIRTUAL]:Botão de Adição de Interfaces - Menu - VirtualA seguinte tela será exibida:Interfaces - Criação de Interface VirtualConfigure-a de acordo com as especificações dos respectivos campos, conforme demonstrado a seguir:\xa0GeneralInterface Virtual – General\xa0Interface: Neste campo de seleção é possível definir a interface desejada. Ex.: eth1;Network Zone: Este campo refere-se à definição do tipo de agrupamento de interfaces. Por padrão temos as zonas “WAN”, “LAN” e “DMZ”. Este é um requisito obrigatório. Ex.: Digite “WAN” e selecione-a na lista;Name: Refere-se à identificação da interface de rede que foi selecionada para a edição. Ex.: “[Eth1v0]”;Description: Defina uma descrição para identificação da interface configurada. Este é um requisito obrigatório. Ex.: “Wan\xa0eth1v0”.\xa0IPv4Interface Virtual – IPv4Para habilitar o padrão do endereçamento IPv4,\xa0marque a caixa de checagem\xa0[] no topo do painel e configure o formulário com os dados correspondentes de cada campo.IP Address: Define o endereço IPv4 da interface virtual. Este é um requisito obrigatório. Ex.: “187.8.187.11”;Mask: Esta caixa de seleção determina a máscara correspondente ao endereçamento IPv4. Este é um requisito obrigatório.\xa0Ex.: “255.255.255.128”;Gateway: Define o endereço de gateway correspondente a rede. Este é um requisito NÃO obrigatório. Ex.: “187.8.187.1”.\xa0IPv6Interface Virtual – IPv6Para habilitar o padrão do endereçamento IPv6, marque a caixa de checagem\xa0[] no topo do painel e\xa0configure o formulário com os dados correspondentes de cada campo.IP Address: Define o endereço IPv6 da interface virtual. Este é um requisito obrigatório. Ex.: “2804:14c:150:29ac::1001”;Prefix: Determina o prefixo correspondente ao endereçamento IPv6. Este é um requisito obrigatório. Ex.: “64”;Gateway:\xa0Define o endereço de gateway correspondente a rede.\xa0Este é um requisito NÃO obrigatório.\xa0AdvancedInterface Virtual – AdvancedEm uma interface Virtual é possível definir o MTU (Maximum Transmission Unit) e alterar o MAC address. Segue detalhes de cada campo:MTU[]:\xa0Para habilitar esta opção, marque a caixa de checagem. Este campo define o MTU da interface virtual, os valores possíveis estão entre 1280 até 9000 (JUMBO FRAME);', 'Atenção, para evitar fragmentação, poderá ser necessário aumentar os valores do MTU.
